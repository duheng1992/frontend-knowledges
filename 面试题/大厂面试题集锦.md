### 1. 请实现 DOM2JSON 一个函数，可以把一个 DOM 节点输出 JSON 的格式

```js
function dom2Json(domtree) {
  let obj = {};
  obj.name = domtree.tagName;
  obj.children = [];
  domtree.childNodes.forEach((child) => obj.children.push(dom2Json(child)));
  return obj;
}

// 迭代实现怎么做
function dom2JsonIterative2(root) {
  if (!root) return null;

  const stack = [{ node: root, parent: null }];
  let result = null;

  while (stack.length > 0) {
    const { node, parent } = stack.pop();

    const obj = {
      name: node.tagName || node.nodeName,
      children: []
    };

    // 跳过空文本节点
    if (node.nodeType === 3 && !node.textContent.trim()) continue;

    if (!parent) {
      result = obj;
    } else {
      parent.children.push(obj);
    }

    // 正序压栈子节点（需要逆序处理）
    const children = Array.from(node.childNodes);
    for (let i = 0; i < children.length; i++) {
      stack.push({ node: children[i], parent: obj });
    }
  }

  return result;
}
```

### 3. 说一下JSON.stringify有什么缺点？

```js
1.如果obj里面有时间对象，则JSON.stringify后再JSON.parse的结果，时间将只是字符串的形式，而不是对象的形式
2.如果obj里有RegExp(正则表达式的缩写)、Error对象，则序列化的结果将只得到空对象；
3、如果obj里有函数，undefined，则序列化的结果会把函数或 undefined丢失；
4、如果obj里有NaN、Infinity和-Infinity，则序列化的结果会变成null
5、JSON.stringify()只能序列化对象的可枚举的自有属性，例如 如果obj中的对象是有构造函数生成的， 则使用JSON.parse(JSON.stringify(obj))深拷贝后，会丢弃对象的constructor；
6、如果对象中存在循环引用的情况也无法正确实现深拷贝；
```

微前端的存在意义可以概括为一句话：  
**“把巨石前端拆小、把异构系统拼整、让团队独立演进”**。具体体现在三方面：

1. 解决“巨石应用”顽疾  
   • 功能膨胀、代码耦合，改一行动全身，发布风险高；微前端把大应用切成多个可独立开发、测试、部署的子应用，降低复杂度与发布成本。  

2. 支持“多团队+多技术栈”并行  
   • 不同团队各管一块，互不阻塞；React/Vue/Angular 甚至旧框架可在同一产品中共存，技术选型自由。  

3. 实现“渐进式迁移”与“历史系统融合”  
   • 不一次性重写旧系统，而是逐步替换或新增模块；新旧功能在运行时无缝聚合，既保护既有投资，又可持续升级。
